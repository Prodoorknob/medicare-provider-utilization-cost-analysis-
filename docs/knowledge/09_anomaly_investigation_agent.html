Why an agent on this dataset

Medicare fraud, waste, and abuse cost an estimated $60–100 B annually. Rule-based screens catch obvious violations but miss the cases that require contextual reasoning. A cardiologist billing 3× the state average might be committing fraud, running a high-volume legitimate practice, or treating a sicker-than-typical panel. Distinguishing those cases requires knowing specialty norms, geographic variation, patient acuity, and the procedural meaning of HCPCS code families — the kind of judgement work where an LLM with structured evidence is meaningfully better than thresholds.

The AllowanceMap silver layer already had the prerequisites: 103 M provider-service rows, 131 specialties, 63 states, 11 years, NPI preserved, HCC risk scores joined. The agent re-uses that infrastructure to identify, contextualize, and triage provider anomalies, then hands the analyst a brief that explains what is suspicious and what is explainable. Specification: PROVIDER_ANOMALY_AGENT_SPEC.md (2026-04-08); built end-to-end 2026-04-23 to 2026-04-24.

Architecture — five-step pipeline

Step 1 Statistical detection anomaly/detect_outliers.py z-score (log1p) + Isolation Forest + temporal rules ↓ Step 2 Contextual retrieval anomaly/retrieve_context.py profiles + benchmarks + top HCPCS + scope check ↓ Step 3 Rule cross-reference anomaly/check_rules.py 10 OIG/CMS-derived fraud indicators (7 evaluable, 3 not) ↓ Step 4 Investigation brief anomaly/generate_brief.py Claude Sonnet 4.6 with cached system prompt ↓ Step 5 Human review gate web/src/app/investigations/ Approve / Escalate / Dismiss + analyst notes

Steps 1–3 are deterministic data operations; the LLM is only invoked at Step 4 to interpret the structured evidence. The orchestrator anomaly/agent.py ranks flagged NPI-years by composite severity, runs Steps 2–4 for each, and writes the brief to local_pipeline/anomaly/briefs/{npi}_{year}.md. Dry-run is the default; --live opts in to API spend.

Phase B — Detection engine

Three independent methods run over npi_profiles.parquet and emit a unified long-format flags.parquet. Each method targets a different failure mode and they are intentionally ORed at the row level — the credible-signal rate comes from the intersection, not the union.

Phase C — Investigation brief generation

For each flagged NPI-year, the orchestrator builds a ProviderContext dataclass containing the provider's metrics, percentile ranks, full year-by-year history, top 10 HCPCS codes (read on demand from the relevant state's silver parquet), benchmark snapshots at national and state levels, and a per-rule RuleCheckResult. That structured evidence is passed to Claude Sonnet 4.6 with a fixed system prompt that defines the analyst persona and the output schema.

The system prompt was originally 1,200 tokens and silently failed to engage Sonnet 4.6's prompt cache (the minimum cacheable prefix on Sonnet 4.6 is 2,048 tokens). In Phase D I padded it to 2,584 tokens with reference material on dataset constraints, HCPCS bucket families, specialty norm heuristics, and per-rule interpretation guidance. cache_read_input_tokens is now non-zero on every call after the first.

Phase D — Specialty scopes, UI, and redaction

Three additions closed the spec out: an empirical specialty-HCPCS scope table, a Next.js review surface, and a runtime NPI-redaction toggle for demos.

Specialty-HCPCS scope table

anomaly/rules/specialty_scopes.py scans silver to build specialty_scopes.parquet — for each specialty, the set of HCPCS codes that are either billed by ≥ 1 % of providers in the specialty OR fall in the cumulative top 99 % of the specialty's service volume. Output: 86,924 rows across 130 specialties; ~27 % of code-specialty pairs are in-scope. Coverage threshold is intentionally generous because most providers legitimately bill an occasional out-of-bucket code (DME supplies, vaccines, transport).

Most interpretively important call: "specialty scope" is defined empirically from what a specialty's population actually bills, NOT from regulatory scope-of-practice rules. Of the 10 regenerated briefs, two trigger OUT_OF_SPECIALTY: an Internal Medicine NPI billing 99.7 % of services on J7170 (a hemophilia biologic) and the multispecialty clinic with 100 % umbrella drift. The notorious optometrist-doing-cataract-surgery case (NPI 1710906219) does NOT trigger OUT_OF_SPECIALTY because 28 % of optometrists in CMS data bill HCPCS 66984 under co-management arrangements. The empirical definition is more honest than a regulatory one and pushes the LLM to do the harder reasoning step (scale and concentration, not just code identity).

Review surface

Next.js routes /investigations (list) and /investigations/[id] (detail) render the briefs sitting in web/public/data/investigations/. web/scripts/sync-briefs.mjs copies briefs from the agent output directory and parses per-rule summaries. The detail page exposes Approve / Escalate / Dismiss buttons and a free-text analyst notes field; decisions persist to localStorage as a placeholder for the eventual feedback-loop / threshold-recalibration pipeline.

NPI redaction

Briefs reference real NPIs from real CMS public data. NPIs are technically public via NPPES, but a list of "providers Claude classified CRITICAL" is a different artifact than a name in NPPES. Redaction is implemented two ways: --mask-npis on the sync script for pre-deploy masking (format 1033****74), and a runtime toggle in the list and detail headers backed by localStorage and synced cross-tab via CustomEvent. NEXT_PUBLIC_REDACT_NPIS=1 locks redaction on for any public deploy.

Phase E — UPCODING and LEIE unlocked

Two rules that started as NOT EVALUABLE stubs became evaluable in this phase: UPCODING via internal pre-computation, and LEIE_EXCLUDED via an external data join. Both use the same pattern as the Phase D specialty-scopes work — a one-pass script produces a sidecar parquet that the retriever loads alongside profiles and benchmarks.

UPCODING — E&M code-level distribution

anomaly/rules/em_distribution.py scans silver for E&M codes and produces two sidecar parquets: em_distributions.parquet (5.94 M NPI-year rows with per-code service counts plus em_est_high_pct = 99214+99215 share of established visits) and em_specialty_benchmarks.parquet (863 specialty-year rows with P50/P75/P90/P95/P99 of the high-tier share). Runtime: 28 seconds on the full 2013–2023 silver.

The rule fires when em_est_high_pct > specialty_P95 on at least 50 established visits. In practice, this is the classic OIG MLN SE1418 upcoding pattern: a provider whose 99214/99215 share sits far above their specialty peers with volume that makes the ratio statistically meaningful.

The saturated-specialty calibration: 68 of 84 analysed specialties have high-tier share saturated at P95 = 100 % (Internal Medicine, Cardiology, Neurology, Psychiatry, etc. — specialties where 99215 is the default established visit). In those specialties the rule literally cannot discriminate, so it reports NOT TRIGGERED with evidence stating "specialty baseline saturated at high-tier." This is the honest behavior; a rule that silently returned FALSE in these cases would be a false negative disguised as a verdict. The rule discriminates meaningfully in 16 specialties with real spread: Orthopedics, Dermatology, Podiatry, Optometry, Urology, Gastroenterology, Ophthalmology, and similar procedural subspecialties.

LEIE_EXCLUDED — OIG exclusion list join

anomaly/external/leie_loader.py downloads the OIG LEIE "UPDATED" CSV from oig.hhs.gov/exclusions/downloadables/UPDATED.csv (~15 MB, 83,001 exclusion records) and writes it to leie_exclusions.parquet. Of those records, 8,375 have valid NPI; the remainder are historical exclusions pre-dating NPI adoption and cannot be joined to CMS data. The retriever indexes by NPI and attaches the exclusion record (type, date, reinstatement status) to every ProviderContext.

An NPI match is a conviction-grade signal under 42 USC 1320a-7: any Medicare payment to an actively excluded NPI is program-ineligible by law. The rule's behaviour reflects that — a match pushes severity to CRITICAL and overrides the statistical risk ranking. Historical (reinstated) exclusions are surfaced as context but do not trigger.

The base-rate observation: 0 of the top 100 composite-severity flags in 2023 were on the active LEIE. This is expected and reassuring — the statistical anomaly detector is finding a different population than the OIG exclusion process. The value of LEIE_EXCLUDED is not in the frequent match (rare by construction) but in ensuring that if an excluded NPI does appear in a flag batch, it cannot slip past the analyst with a LOW classification.

Validation — 100 briefs on 2023

The validation run ranks the top 100 composite-severity NPI-years in 2023 and regenerates briefs through the updated retriever and rule engine. Every brief now evaluates all 10 rules, including the two previously-stub UPCODING and LEIE_EXCLUDED. Reading a representative HIGH-risk brief (NPI 1023087285, Gastroenterology, PA, 2023) shows the model correctly distinguishing vedolizumab biologic unit-billing from phantom billing and pushing the case to HIGH rather than CRITICAL, while flagging the 96365/J3380 administration-code ratio for manual follow-up in data limitations — exactly the kind of reasoning that rule thresholds alone cannot produce.

Showcase finding — NPI 1710906219, Optometry, OH, 2018

The single most defensible artifact for "the LLM is doing real work" comes from the top-severity brief.

What the LLM did with that evidence: identified HCPCS 66984 as a code optometrists can legitimately bill in some states under co-management, but called the scale "entirely different" from any co-management convention. Noted the $1.38 avg allowed as evidence Medicare may already be denying or severely discounting these claims. Recommended five specific actions including immediate referral to the Ohio Zone Program Integrity Contractor, claim-level audit for 2018–2021 with date-of-service and rendering-provider fields, identifying the facility associated with the 100 % facility billing setting, cross-referencing the 12 beneficiaries against ophthalmologist claims, and verifying the provider's enrollment record for any group-affiliation change in 2018 that explains the structural shift.

OUT_OF_SPECIALTY did not trigger on this NPI — co-management billing is common enough that the empirical scope set includes 66984 for optometry. The CRITICAL classification came from HIGH_INTENSITY plus PROCEDURE_CONCENTRATION plus the temporal context, with the LLM tying them together. This is exactly the case the spec's Step 4 was designed to surface: a pattern that no single rule can convict on but that becomes obvious when the evidence is read jointly.

Calibration check — the MEDIUM that should have been MEDIUM

NPI 1386678977 (Multispecialty Clinic / Group Practice, MI, 2013) sat at composite-severity rank 3 but received a MEDIUM (42/100) classification rather than HIGH or CRITICAL. The brief's reasoning — that the elevated avg_allowed is a "specialty-mismatch artifact" because the practice billed pure radiation oncology codes under a multispecialty group designation, that the 2.45 HCC risk score is legitimate for an oncology panel, and that no formal fraud rules triggered — is the calibration check the spec called for. A naive scoring system that summed flag severity would have classified this CRITICAL. The LLM resisted because the evidence taken jointly is not actually suspicious.

What this proves about the dataset

ClaimEvidence
Silver has enough resolution to support meaningful peer comparisons26.8 % thin (state, specialty, year) groups force fallback to (specialty, year), but the fallback works — rare specialties like Advanced Heart Failure (783 providers) still surface coherent benchmarks.
Empirical specialty scope is more useful than regulatory scopeOptometrists legitimately bill 66984 28 % of the time under co-management. A regulatory scope rule would over-fire on this; the empirical rule lets the LLM decide whether the scale is consistent with co-management or not.
Risk-score-aware reasoning is necessary, not optionalThe MEDIUM brief explicitly invoked HCC = 2.45 to argue the high avg_allowed was clinically defensible. Without the join from pull_provider_data.py, the agent could not distinguish "sicker panel" from "fraud."
Composite (multi-method) signal is the right detection unitSingle-method flag rates (1–2 % each) compound to ~7 % on union but intersect to 0.77 % on ≥ 2 methods. The intersection is where false-positive rate drops to a triage-friendly level without threshold tightening.

Per-role things to volunteer before being asked

Data Engineer interview

  • NPI was preserved at silver but dropped at gold — this agent depended on the silver layer surviving the feature-engineering step
  • The path-inheritance bug from the gold script (../data) silently produced 0 % risk-score coverage on first run; caught by smoke test before scaling
  • Specialty-HCPCS scope is computed once with a streaming pass; per-state silver sharding made this trivially parallelizable
  • Empirical scope vs regulatory scope is the right call given that scope-of-practice varies by state and the data does not encode it

ML Engineer interview

  • Naive z-score fired at 5× the spec target on lognormal billing data; log1p-on-heavy-tailed-metrics brought it back to ~1 %
  • Composite (≥ 2 methods) signal is the calibration unit; raw OR is not actionable at 7 % flag rate
  • System prompt cached only after padding to 2,048-token Sonnet 4.6 minimum; first run silently no-cached and I caught it in the usage telemetry
  • Top-flagged specialties (Pain Management, Radiation Oncology, Diagnostic Radiology) match OIG fraud priorities — that is the sanity check, not synthetic-data evals
  • Three detection methods are intentionally independent: z-score is univariate, Isolation Forest is multivariate, temporal is sequence-aware

Data Scientist interview

  • Sample MEDIUM brief is the calibration check — the LLM correctly resisted convicting on a specialty-mismatch artifact
  • OUT_OF_SPECIALTY does NOT trigger on the optometrist case because 28 % of optometrists bill 66984 — the empirical definition forces the model to reason about scale, not just code identity
  • Three of the ten rules are structurally not evaluable on public CMS data (IMPOSSIBLE_DAY, UNBUNDLING, BENEFICIARY_SHARING); brief discloses this every time. The original fourth — UPCODING — was unlocked in Phase E via a one-pass E&M distribution pre-compute
  • HCC risk score is the single most important context variable for distinguishing "sicker panel" from "fraud" — without it, intensity rules over-flag oncology, nephrology, hospice
  • Cost: $0.04 per brief on Sonnet 4.6 with caching; Opus 4.7 was not necessary for this task

Top-5 design decisions

  • 1Composite signal (≥ 2 methods fire) over single-rule thresholds. Single-rule flag rates compound badly; intersection is where credible-signal rate lives.
  • 2Empirical specialty scope, not regulatory. The optometrist-billing-cataract-surgery case is exactly the wrong thing for a regulatory scope rule to flag, because co-management is real. Force the LLM to do the scale-and-concentration reasoning.
  • 3Disclose what was NOT checked in every brief. Three of the ten rules are structurally not evaluable; pretending otherwise is dishonest. The "Data Limitations" section is part of the contract.
  • 4Dry-run as the default; --live opts in to API spend. The harness should not auto-cost the user's account; the cost boundary deserves a flag.
  • 5Padded system prompt to engage Sonnet 4.6's 2,048-token cache minimum. The first 10-brief run cost $0.36 with no caching; subsequent runs are cheaper and the prompt itself got better because the padding is real domain context, not filler.

30-second pitch

Phase 9 ships a fraud-investigation agent on top of the AllowanceMap silver layer. Statistical detection (z-score, Isolation Forest, temporal) flags 0.77 % of NPI-years on a multi-method intersection; for each, a contextual evidence package — provider metrics, percentile ranks, year-by-year history, top HCPCS codes, specialty benchmarks, empirical scope check, E&M distribution, OIG LEIE match — is passed to Claude Sonnet 4.6 with a cached 2,584-token system prompt that defines the analyst persona and the output schema. Briefs are analyst-grade: the top brief identified an optometrist billing 1,784 cataract surgeries to 12 beneficiaries, hypothesized NPI identity theft from the $1.38 avg allowed pattern, and recommended five concrete next steps. Seven of ten rules are now evaluable (LEIE_EXCLUDED and UPCODING unlocked in Phase E); the remaining three are structurally blocked by CMS public data scope, and the brief discloses that every time. Cost is $0.04 per brief with caching, and the briefs are reviewable on a Next.js /investigations page with Approve / Escalate / Dismiss controls.
